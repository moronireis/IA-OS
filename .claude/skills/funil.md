# Skill /funil - Design de Funil Completo

Quando o usuario invocar `/funil`, siga este processo:

## 1. Coleta de Informacoes

Pergunte:
- Tipo de funil desejado: lancamento (semente/classico/pago), perpetuo, webinar, e-book, high ticket, hibrido
- Nicho e avatar
- Produto/servico e faixa de preco
- Orcamento de trafego
- Tech stack atual (Hotmart? WordPress? Active Campaign? etc.)
- Objetivo principal (leads, vendas, nurturing, upsell)

## 2. Consultar Knowledge Base

Leia arquivos relevantes de TODOS os dominios conforme o tipo:
- Lancamento: `01-lancamentos/` (FL, Baldan, Tabari)
- High Ticket: `02-high-ticket/fht-masterplan.md` (SLG → 1a1 → AR)
- Trafego: `03-trafego/` (campanhas, audiencias)
- E-book: `04-conteudo-copy/ebooks-webinars.md` (R$9.90, R$1 CAC)
- Webinar: `04-conteudo-copy/ebooks-webinars.md` (R$12M+ framework)

## 3. Gerar Diagrama do Funil

Entregue:
- **Diagrama visual** (texto) do funil: awareness → consideracao → conversao → retencao
- **Paginas necessarias**: cada URL com proposito (/cadastro, /obrigado, /vendas, etc.)
- **Emails/automacoes**: sequencia completa com timing
- **Metricas-alvo por etapa**: taxa conversao esperada em cada ponto
- **Tech stack recomendada**: ferramentas por funcao
- **Estimativa de resultados**: projecao baseada em benchmarks reais

## 4. Benchmarks por Tipo de Funil

| Funil | CAC | Conversao | ROAS |
|-------|-----|-----------|------|
| E-book (R$9.90) | R$1/cliente | 18% order bump | Alto (Scale case) |
